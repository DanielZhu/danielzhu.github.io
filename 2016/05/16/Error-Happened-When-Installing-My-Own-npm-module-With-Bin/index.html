Error: [env: node : No such file or directory]
npmjsnodejs
16/05/16 21:05

当我们从其它平台迁移过来继续开发项目时，有时候经常会遇到一些因为编码不同而导致的问题

在开发npm模块时，遇到了个

在本地新建的项目，文件夹结构是从之前windows的PC上直接copy过来，在执行npm install * 时，就总是会提示

也真的是醉了呀，试了那个旧的pc的module，还是可以通过npm install来安装的，只是在执行cli时，会报错。。。 env: node\r: No such file or directory 一查是因为编码的不同而导致的问题，

参考以下帖子：

  • StackOverflow | Node script executable not working on Mac : env: node\r: No such file or directory
  • Github | env: node\r: No such file or directory #75

于是考虑重新更换成unix编码格式保存旧的项目文件 ./bin/***，卸载后重新安装，搞定，能够执行 *** --version 拿到版本号。

思路迁移回来，考虑到新的npm模块的规模很小，所以手动删掉了总是报错找不到文件的那个 /bin/A 的文件A，retry install，继续失败

最终解决方案：

重新利用 npm init 来新建新项目，当然代码还是从有问题的module里 copy & paste， sublime text操作一切，最后再retry install，终于不再报错了！！！浪费了昨天凌晨2个小时在那研究到底哪儿出问题了。。。不就是个简单的在原本npm module里支持command么。。。醉了。。。
